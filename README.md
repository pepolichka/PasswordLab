# PASSWORD LAB — Генератор и проверка надёжности паролей

**PASSWORD LAB** — учебный веб-проект для курсовой работы.
Приложение генерирует пароли, проверяет их по базовым правилам безопасности, оценивает надёжность и хранит историю генерации в браузере.

Проект сделан на чистых **HTML + CSS + Vanilla JavaScript** без фреймворков и сборки, поэтому его легко объяснить на защите и удобно публиковать на GitHub.

## Возможности

- Генератор паролей:
  - длина от 6 до 64;
  - наборы символов: строчные, заглавные, цифры, спецсимволы;
  - опции: исключить похожие символы (`O/0`, `I/l/1`), без повторов;
  - безопасная случайность через `crypto.getRandomValues`.
- Проверка пароля:
  - длина, регистр, цифры, спецсимволы;
  - проверка по локальному списку популярных паролей (200 значений);
  - рекомендации по улучшению.
- Оценка надёжности:
  - шкала 0–100;
  - статусы: `Слабый / Средний / Надёжный`;
  - учёт бонусов и штрафов (длина, разнообразие, повторы, последовательности, слабые слова).
- История генерации:
  - хранение последних 10 паролей в `localStorage`;
  - маскирование паролей в списке;
  - отметка даты и времени;
  - кнопка очистки.
- Визуальный стиль:
  - тёмный `Matrix / dark net / hacking` интерфейс;
  - двухслойный фон с падающим кодом через два `<canvas>`;
  - размытый «глубокий» фон, не мешающий кликам и чтению.

## Запуск

### Вариант A: обычное открытие файла

1. Склонировать репозиторий:
   ```bash
   git clone <your-repo-url>
   cd PasswordLab
   ```
2. Открыть файл `index.html` двойным кликом или через браузер.

### Вариант B: локальный сервер

1. В корне проекта выполнить:
   ```bash
   python3 -m http.server
   ```
2. Открыть в браузере адрес `http://localhost:8000`.

## Как устроена генерация

1. Пользователь выбирает длину и параметры в блоке генератора.
2. Формируется алфавит из отмеченных наборов символов.
3. Символы выбираются через `crypto.getRandomValues` (без `Math.random`).
4. В режиме «без повторов» выбранный символ удаляется из временного пула.
5. При некорректной конфигурации выводится понятная ошибка.

## Как считается надёжность

Модуль `js/strength.js` возвращает итоговый результат от 0 до 100:

- бонусы за длину и разнообразие типов символов;
- бонус за долю уникальных символов;
- штраф за повторы;
- штраф за простые последовательности (`123`, `qwe`, `asd`);
- штраф за слабые слова (`password`, `admin`, `qwerty`).

Итог ограничивается диапазоном `0..100`.
